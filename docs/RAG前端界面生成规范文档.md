## 1. 设计原则

### 1.1 核心目标

- **降低认知负荷**: 将复杂的检索-生成流程可视化，让用户直观理解"从哪来"（检索）和"怎么说"（生成）。
- **即时反馈**: 任何操作（上传、分块、索引、检索）都应在 300ms 内给出状态反馈。
- **可调试性**: 提供透明化的检索中间结果，支持开发者/运营人员快速定位"幻觉"来源。

### 1.2 设计范式

- **双栏布局为主**: 左侧知识库/文档管理，右侧对话/检索测试区。
- **卡片化信息**: 文档、Chunk、引用来源均以卡片形式呈现，支持展开/折叠详情。
- **状态显性化**: 用颜色+图标+文字三重编码表示系统状态（如索引中、已就绪、异常）。

## 3. 全局组件规范

### 3.1 顶部导航栏 (Navbar)

- **高度**: `56px`
- **元素**: Logo、当前知识库选择器（下拉）、全局搜索框、用户头像菜单
- **知识库选择器**: 显示当前 KB 名称+状态圆点；切换时保留当前页面上下文（如从 `/kb/1/docs` 自动跳转 `/kb/2/docs`）

### 3.2 侧边栏 (Sidebar)

- **宽度**: `240px`（可折叠至 `64px`）
- **菜单项**: 图标 + 文字；当前项高亮（`primary` 色背景，透明度 8%）
- **底部区域**: 固定显示系统状态（如 "索引服务: 正常" / "模型 API: 延迟 120ms"）

### 3.3 全局状态指示器

- **Toast 通知**: 操作成功/失败反馈，自动消失（3s），错误类手动关闭。
- **全局进度条**: 顶部细线（`2px`），用于文档批量上传、大规模索引等长耗时任务。
- **WebSocket 状态徽章**: 右下角固定小圆点，绿色（已连接）/ 黄色（重连中）/ 红色（断开）。

## 4. 核心页面规范

### 4.1 对话界面 (Chat) — 终端用户入口

#### 布局

- **全屏布局**: 无侧边栏，仅顶部导航 + 中央对话区 + 底部输入框。
- **对话区**: 宽度最大 `800px`，居中，上下留白 `24px`。

#### 消息气泡规范

| 类型     | 背景色                     | 圆角                | 最大宽度 | 特殊元素               |
| -------- | -------------------------- | ------------------- | -------- | ---------------------- |
| 用户消息 | `surface-container` (灰) | `16px` (左下直角) | 70%      | 无                     |
| AI 消息  | `surface` (白)           | `16px` (右下直角) | 85%      | 引用来源角标、反馈按钮 |

#### 引用来源 (Citation)

- **展示位置**: AI 消息正文下方，以 `[1]`, `[2]` 角标形式嵌入文本；点击角标展开右侧/底部抽屉。
- **来源卡片**: 显示文档名、页码/段落、相似度分数、原文摘录（最多 3 行，可展开）。
- **溯源链接**: 点击卡片可跳转至原文预览（高亮对应文本）。

#### 输入框

- **高度**: 自适应，最小 `56px`，最大 `200px`（多行文本）。
- **功能按钮**: 附件上传（图片/文档）、发送、语音输入（可选）。
- **占位符文案**: "请输入问题，系统将基于知识库回答..."
- **快捷键**: `Enter` 发送，`Shift+Enter` 换行。

#### 空态 (Empty State)

- 中央展示知识库图标 + 欢迎语 + 3-5 个预设问题胶囊（Quick Actions），点击直接发送。

### 4.2 知识库管理 (Knowledge Base)

#### 列表视图

- **卡片网格**: 每行 3-4 张卡片（响应式）。
- **卡片内容**: KB 名称、文档数量、索引状态、创建时间、操作按钮（设置/删除）。
- **状态标签**:
  - `未索引` (灰色)
  - `索引中` (蓝色，带旋转图标)
  - `已就绪` (绿色)
  - `异常` (红色，悬停显示错误详情)

#### 创建流程

1. **步骤条**: 基本信息 → 解析配置 → 确认创建。
2. **基本信息**: 名称、描述、可见性（公开/私有）。
3. **解析配置**: 选择解析器（通用文本/Markdown/PDF/表格）、分块策略（自动/固定长度/语义）、Embedding 模型。
4. **确认页**: 配置摘要，创建后自动跳转文档上传页。

### 4.3 文档管理 (Documents)

#### 上传区域

- **拖拽上传**: 虚线边框区域，支持多文件；显示文件类型限制（如 `.pdf, .docx, .txt, .md`）。
- **上传列表**: 文件名、大小、进度条、状态（上传中/解析中/已完成/失败）。
- **批量操作**: 勾选后顶部出现浮动工具栏（删除、重新索引、移动）。

#### 文档列表

- **表格视图**: 文件名、页数/字数、上传时间、索引状态、操作。
- **展开详情**: 点击行展开，显示分块数量、解析日志、元数据（Metadata）。

### 4.4 分片管理 (Chunks)

#### 双栏布局

- **左侧**: 分片列表（显示前 80 字符摘要 + 序号）。
- **右侧**: 选中分片的完整内容编辑区。

#### 分片卡片

- **元数据展示**: 所属文档、页码、Token 数、向量 ID（可点击复制）。
- **编辑功能**: 支持修改文本内容（重新触发 Embedding）、调整元数据、合并/拆分 Chunk。
- **搜索过滤**: 顶部全文搜索框，支持按文档筛选。

### 4.5 检索测试 (Retrieval Test) — 开发者调试

#### 布局

- **左右分栏**: 左侧查询输入 + 参数配置，右侧结果列表。

#### 查询区

- **输入框**: 多行文本，支持输入问题。
- **参数面板**:
  - Top-K: 滑块（1-20，默认 5）
  - 相似度阈值: 滑块（0-1，默认 0.7）
  - 检索模式: 单选（向量检索/全文检索/混合检索/重排序 Rerank）
  - 元数据过滤: JSON 编辑器（可选）

#### 结果区

- **结果列表**: 每个结果展示:
  - 排名序号
  - 相似度分数（带颜色梯度：高绿-中黄-低红）
  - 来源文档 + Chunk 摘要
  - 高亮匹配关键词
- **对比模式**: 支持保存多次查询结果，进行 A/B 对比。

### 4.6 系统设置 (Settings)

#### 配置分组

- **模型配置**: LLM 选择（OpenAI/Claude/本地模型）、API Key 管理（加密输入）、温度参数、最大 Token。
- **Prompt 模板**: 系统提示词编辑器、上下文模板变量（如 `{context}`, `{question}`）自动补全。
- **检索配置**: 默认 Top-K、默认阈值、是否启用重排序模型。
- **安全与权限**: 访问白名单、敏感词过滤、速率限制。

### 4.7 数据分析 (Analytics)

#### 仪表盘

- **核心指标卡片**: 总对话数、平均引用率、平均响应时间、Token 消耗（今日/本周/本月）。
- **趋势图表**: 对话量折线图、用户满意度柱状图。
- **日志表格**: 时间、用户、问题、回答摘要、引用数量、操作（查看详情）。
- **详情抽屉**: 单条对话的完整回溯，包括检索到的 Chunk、LLM 原始请求/响应（JSON 可折叠）。

## 5. 交互与动效规范

### 5.1 状态转换

- **索引中动画**: 卡片上的状态图标使用 `spin` 旋转动画；文档列表行首添加脉冲圆点。
- **生成中动画**: AI 消息气泡底部显示"正在思考..." + 三个跳动圆点；检索阶段显示"正在查找相关资料..."（进度条可选）。
- **打字机效果**: AI 回复内容逐字显示，速度 `20ms/字`，支持用户点击"跳过"立即显示全文。

### 5.2 微交互

- **Hover 效果**: 卡片阴影加深（`shadow-sm` → `shadow-md`），按钮背景色加深 5%。
- **点击反馈**: 按钮点击时缩放 `0.98`（`transform: scale(0.98)`），持续 `100ms`。
- **拖拽反馈**: 上传区域拖拽进入时边框变为实线 `primary` 色，背景添加 `primary` 色 5% 透明度。

### 5.3 加载态

- **骨架屏**: 列表页使用骨架屏（Skeleton），避免整页 Loading。
- **局部加载**: 按钮内嵌圆形进度条（`16px`），操作进行中禁用按钮并替换文字为"处理中..."

## 7. 数据与状态管理规范

### 7.1 前端状态层

- **全局状态 (Zustand/Redux)**:
  - `currentKb`: 当前选中知识库 ID
  - `user`: 用户信息、权限
  - `systemStatus`: 各服务健康状态
- **会话状态 (React Context/Local State)**:
  - `chatHistory`: 当前对话消息列表
  - `retrievalParams`: 检索测试参数
- **服务端状态 (React Query/SWR)**:
  - 文档列表、Chunk 列表、知识库列表、分析数据（均带缓存、自动刷新、乐观更新）

### 7.2 API 交互规范

- **流式响应 (SSE)**: 对话接口必须支持 `EventSource`，前端逐段接收并渲染；中断时发送 `abort` 请求。
- **错误处理**: HTTP 状态码统一处理（401 跳转登录，403 显示权限不足，500 显示友好错误页）。
- **分页**: 列表接口统一使用 Cursor/Offset 分页，前端滚动加载或页码器。

### 7.3 文件处理

- **上传前校验**: 文件类型、大小（前端+后端双重校验）。
- **分片上传**: 超过 `10MB` 的文件自动触发分片上传，显示总进度。
- **预览**: PDF/图片支持前端预览（PDF.js / 原生 img），文本文件直接读取前 1000 字符预览。

## 8. 响应式适配规范

### 8.1 断点定义

- **Desktop**: `>= 1280px` — 完整双栏/多栏布局
- **Tablet**: `768px - 1279px` — 侧边栏折叠为图标，部分页面改为单栏

### 8.2 移动端特殊规范

- **对话页**: 全屏消息区，输入框固定底部，键盘弹出时消息区自动滚动至底部。
- **导航**: 底部 Tab 栏替代侧边栏（Chat / KB / Settings）。
- **触摸优化**: 按钮最小点击区域 `44x44px`，支持左右滑动切换 Tab。

## 9. 可访问性 (Accessibility) 规范

- **键盘导航**: 所有交互元素支持 `Tab` 聚焦；弹窗支持 `Esc` 关闭；抽屉支持 `Enter` 确认。
- **屏幕阅读器**: 图片必须有 `alt` 文本；状态变化使用 `aria-live` 区域通知；表格使用语义化 `<table>` 标签。
- **色彩对比**: 正文与背景对比度 >= 4.5:1；大号文本 >= 3:1。
- **焦点样式**: 聚焦环使用 `2px solid primary` + `2px offset`，确保可见性。

## 11. 附录: 命名规范

### 11.1 文件/目录命名

- 页面组件: `PascalCase.tsx` (如 `ChatPage.tsx`)
- 通用组件: `PascalCase/index.tsx` (目录形式)
- 工具函数: `camelCase.ts` (如 `formatDate.ts`)
- 样式文件: 跟随组件名，使用 Tailwind 时尽量减少独立 `.css` 文件

### 11.2 接口命名

- API 函数: `fetch[Resource][Action]` (如 `fetchKbList`, `uploadDocument`)
- 类型定义: `I[Name]` 或 `[Name]Type` (如 `IKnowledgeBase`, `ChunkType`)
- 枚举: `E[Name]` 或 PascalCase (如 `EIndexStatus`)

### 11.3 路由命名

- 全小写，连字符分隔（kebab-case），语义化（如 `/retrieval-test` 而非 `/rt`）
